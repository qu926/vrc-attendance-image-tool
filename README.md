# VRCイベント出席画像ツール

VRCイベントの出席メンバーを「インスタンス1」「インスタンス2」に分けて選び、X投稿用のPNG画像を書き出す静的Webツールです。

依存関係なしで使えます。`index.html` をブラウザで開くだけで動作します。

## できること

- メンバー画像を後から複数追加
- 指定済みの初期メンバー20名を同梱済み
- ファイル名から自動で名前を作成し、後から名前編集
- 画像下に出す「前半のみ」「後半のみ」などのメンバー別メモを編集
- 未配置、インスタンス1、インスタンス2へ簡単に振り分け
- 背景画像、タイトル、日付、補足メモを設定
- 1600x900、1200x675、1080x1080、1080x1350 の画像サイズに対応
- 全体PNGとインスタンス別PNGの書き出し
- JSONバックアップと復元
- ブラウザのlocalStorageに自動保存
- Vercel KV APIを使った共有DB保存

## 使い方

1. `index.html` をブラウザで開きます。
2. 初期メンバー20名が「未配置」に登録されています。
3. 必要に応じて「メンバー画像」から新人・追加参加者の画像を追加します。
4. 登録メンバーのカードで名前と表示メモを必要に応じて直します。PNGでは画像下に名前ではなく表示メモが出ます。
5. 各カードの `1` または `2` を押して、参加するインスタンスに振り分けます。
6. タイトル、日付、背景、テンプレート、配置を調整します。
7. 右側のプレビューを確認し、「全体PNGを保存」を押します。
8. インスタンスごとに分けた画像が必要な場合は、「1インスタンスPNG」または「2インスタンスPNG」を押します。各インスタンスは10〜20人程度で変動しても、現在の振り分け内容に合わせてPNGを書き出せます。

作業状態を共有・退避したい場合は「JSON書き出し」を使ってください。メンバー別の表示メモも保存され、別のPCや別のブラウザでは「JSON読み込み」で復元できます。

## 共有DBで別PCと同期する

GitHub Pagesだけではサーバー処理や秘密鍵を置けないため、共有DBを使う場合はVercel KVのAPIを併用します。予約管理ツールと同じVercelアカウントにこのリポジトリを追加し、Vercel KVを接続してください。

1. VercelでこのGitHubリポジトリをImportします。
2. Vercel StorageでKVを作成し、プロジェクトに接続します。
3. `KV_REST_API_URL` と `KV_REST_API_TOKEN` がEnvironment Variablesに入っていることを確認します。
4. デプロイ後のVercel版ツールを開きます。
5. まず1台目で「共有DBへ保存」を押し、別PCでは「共有DBから読込」を押します。

共有DBの接続先はアプリ内で固定されているため、利用者が共有API URL、共有ID、合言葉を入力する必要はありません。APIの秘密トークンはVercel側の環境変数にだけ置き、`index.html` や `app.js` には書かないでください。

GitHub Pages版には `/api/data` がないため、共有DBを使う場合はVercel版を使ってください。GitHub Pages版では共有DBボタンと自動同期は無効になります。自動同期をオンにすると、編集後に共有DBへ保存します。同時編集は共同編集ではなく、最後に保存した内容が優先されます。画像はData URLとして保存されるため、画像が多い場合は通信量とDB容量が増えます。

## GitHub Pagesで公開する

静的HTMLとして動くため、GitHub Pagesにそのまま公開できます。

1. GitHubで新しいリポジトリを作成します。
2. このフォルダ内のファイルをコミットしてpushします。
3. GitHubのリポジトリ画面で `Settings` を開きます。
4. `Pages` を開き、`Source` を `Deploy from a branch` にします。
5. `Branch` に `main`、フォルダに `/root` を選んで保存します。
6. 表示されたGitHub PagesのURLをイベント運営メンバーに共有します。

## exe化する

Windows向けexeが必要な場合は、任意でElectronを使えます。通常利用ではインストール不要です。

```bash
npm install
npm run start
npm run pack
npm run dist
```

- `npm run start`: Electronでローカル起動
- `npm run pack`: インストーラーなしの配布用フォルダを作成
- `npm run dist`: Windows向けインストーラーやportable exeを作成

生成物は `dist/` に出力されます。
